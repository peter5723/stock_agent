# 项目逻辑说明（Agent / SFT / 测评）

## 1. 项目目标

这套项目围绕 A 股分析做了三条主链路：

- Agent 推理链路：用 ReAct Agent 调用四类数据工具，生成研报
- SFT 训练链路：把金融情绪数据转成 ChatML，训练 Qwen2.5-7B LoRA
- 测评链路：自动评估工具调用成功率、数据一致率、端到端时延与质量分

---

## 2. Agent 逻辑

### 2.1 入口与职责

- 入口文件：`multi_agent.py`
- 工具实现：`tools_mcp.py`
- 本地模型服务：`open_vllm.sh`（vLLM OpenAI 兼容接口）

### 2.2 执行流程

1. `multi_agent.py` 初始化 LLM（指向 `http://localhost:8000/v1`）
2. 注册四个工具：
   - `get_fundamental_data`（基本面）
   - `get_technical_data`（技术面）
   - `get_valuation_data`（估值面）
   - `get_news_sentiment`（新闻情绪）
3. 通过 `create_react_agent(llm, tools)` 创建 Agent 执行器
4. Agent 按 ReAct 循环调用工具并收集 Observation
5. 输出最终 Markdown 研报，保存为 `Report_<stock>.md`

### 2.3 工具层的稳定性设计（tools_mcp.py）

- BaoStock 支持重试与可选 SOCKS5 代理
- AkShare 作为兜底数据源（BaoStock 失败时自动降级）
- 新闻情绪工具支持 JSON 清洗与降级中性结果
- 引入短 TTL 缓存，减少重复请求延迟

---

## 3. SFT 逻辑

### 3.1 数据准备链路

- 文件：`prepare_sft_data.py`
- 输入：`FinGPT/fingpt-sentiment-train`
- 输出：`qwen_finance_sft.jsonl`

处理步骤：

1. 读取原始金融情绪样本
2. 将标签映射为结构化输出：
   - `sentiment`
   - `score`
   - `risk_tags`
3. 组装为 ChatML 三段消息（system / user / assistant）
4. 落盘为 JSONL 供训练使用

### 3.2 训练链路

- 文件：`train_qwen_sft.py`
- 基座模型目录：`./qwen_model`
- LoRA 输出目录：`./saves/qwen2.5-7b-finance-lora`

训练流程：

1. 加载 tokenizer 与 base model
2. 注入 LoRA（attention + MLP 关键层）
3. 用 `apply_chat_template` 将 messages 转为训练文本
4. 通过 `Trainer` 做 Causal LM 训练并保存 LoRA 权重

### 3.3 推理部署链路

- 文件：`open_vllm.sh`
- 关键参数：
  - `--enable-lora`
  - `--lora-modules finance_news=./saves/qwen2.5-7b-finance-lora`
  - `--enable-auto-tool-choice`
  - `--tool-call-parser hermes`

---

## 4. 测评逻辑

### 4.1 入口与输出

- 入口文件：`eval_agent.py`
- 输出文件：
  - `{prefix}_results.csv`（逐 case 明细）
  - `{prefix}_summary.json`（聚合指标）

### 4.2 单 case 评测流程

1. 首轮调用模型，拿到 `tool_calls`
2. 执行工具调用（支持并发，代理模式自动降为串行）
3. 二轮调用模型汇总最终研报
4. 计算指标并记录

### 4.3 指标体系

- 工具调用成功率：`tool_success_rate`
- 数据一致率：`data_consistency_rate`
- 端到端时延：`e2e_latency_s`（并统计 P50/P90/P99）
- 质量评分：
  - 客观规则分（40）
  - 可选主观评审分（60）

### 4.4 异常与降级

- 若第二轮汇总失败，评测会降级生成报告并记录：
  - `degraded_e2e = true`
  - `error_stage = second_completion`
- 这样可避免“工具全成功但端到端被整体判失败”的误判

---

## 5. 建议运行顺序

1. 先准备训练数据：`prepare_sft_data.py`
2. 训练 LoRA：`train_qwen_sft.py`
3. 启动 vLLM：`open_vllm.sh`
4. 单次 Agent 验证：`multi_agent.py`
5. 批量评测：`eval_agent.py`

---

## 6. 你可以优先关注的文件

- `multi_agent.py`：Agent 主编排
- `tools_mcp.py`：工具实现、代理、缓存、降级
- `prepare_sft_data.py`：数据转换
- `train_qwen_sft.py`：SFT + LoRA 训练
- `open_vllm.sh`：推理服务启动
- `eval_agent.py`：评测框架与指标输出
